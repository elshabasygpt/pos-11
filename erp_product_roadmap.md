## 3. 📖 User Stories الرئيسية

### نقطة البيع
> **"كمحاسب عند الكاونتر، أريد إتمام عملية البيع في أقل من 30 ثانية باستخدام الباركود، بدون الحاجة لإدخال يدوي، لأن العميل لا ينتظر."**

**Acceptance Criteria:**
- ✅ بحث بالباركود يظهر المنتج فوراً
- ✅ حساب VAT 15% تلقائي
- ✅ قبول دفع متعدد (نقد + بطاقة)
- ✅ طباعة فاتورة ZATCA خلال 2 ثانية

### إدارة المخزون
> **"كمدير مستودع، أريد تنبيهاً فورياً عند وصول أي منتج لحد الحد الأدنى، لأتمكن من إعداد أمر شراء قبل النفاد."**

**Acceptance Criteria:**
- ✅ تنبيه داخل النظام + إشعار
- ✅ قائمة المنتجات قارب نفادها مع كميتها الحالية والمطلوبة
- ✅ زر "إنشاء أمر شراء" مباشرة من التنبيه

### التقارير
> **"كمالك شركة، أريد رؤية ربحي الحقيقي اليوم، بدون الحاجة لانتظار المحاسب لساعات."**

**Acceptance Criteria:**
- ✅ Dashboard يحمل خلال 3 ثواني
- ✅ المبيعات، المشتريات، صافي الربح — كلها على شاشة واحدة
- ✅ مقارنة مع الشهر الماضي ونسبة النمو

### ZATCA Compliance
> **"كمسؤول مالي، أريد إصدار فاتورة إلكترونية موافقة لمتطلبات هيئة الزكاة بدون أي تدخل يدوي."**

**Acceptance Criteria:**
- ✅ QR Code ZATCA على كل فاتورة
- ✅ XML بتنسيق UBL 2.1
- ✅ إرسال تلقائي لـ FATOORA API
- ✅ تقرير VAT جاهز للتقديم ربع السنوي

## 4. 🎨 اقتراحات تحسين تجربة المستخدم (UX)

### المشاكل الشائعة في أنظمة ERP الحالية وكيف نحلها:

| المشكلة الشائعة | حلنا |
|---|---|
| واجهات معقدة ومربكة | **Dashboard يعرض 5 أرقام فقط** — الأكثر أهمية |
| الحاجة لتدريب أسابيع | **Onboarding Wizard** خطوة بخطوة عند أول تسجيل |
| تقارير بطيئة | **كل التقارير تحمل في < 3 ثواني** (Optimized Queries) |
| صعوبة على الجوال | **Mobile-First Design** + PWA |
| خطأ في الإدخال | **Smart Defaults** + Validation فوري |
| فقدان البيانات | **Autosave + Offline Mode** لنقطة البيع |

### قواعد UX ذهبية للنظام:
1. **القاعدة 3 نقرات:** أي عملية أساسية لا تتجاوز 3 نقرات
2. **Search First:** بحث عالمي في أعلى الصفحة (Ctrl+K)
3. **Smart Notifications:** تنبيهات ذكية فقط — لا spam
4. **Dark Mode:** ضروري لمحاسبي الليل!
5. **Keyboard Shortcuts:** لمستخدمي POS السريعين

## 6. 🗺️ Roadmap التنفيذ

### 🔴 Phase 1 — MVP (الشهر 1-2)
**الهدف:** نظام تجاري أساسي يعمل بالكامل

```
الأسبوع 1-2:
  ✅ Setup & Infrastructure (Multi-tenant, Auth, RBAC)
  ✅ Database Schema (Central + Tenant)
  ✅ API Foundation

الأسبوع 3-4:
  ✅ Inventory Module (Products + Warehouses)
  ✅ CRM Module (Customers + Suppliers)
  
الأسبوع 5-6:
  ✅ Sales Module (Invoices + POS)
  ✅ Purchases Module (Purchase Invoices)
  ✅ Basic Accounting (Double-Entry Auto)

الأسبوع 7-8:
  ✅ ZATCA Integration (Phase 2 Compliance)
  ✅ Dashboard + Basic Reports
  ✅ Testing + Bug Fixes
  🚀 LAUNCH MVP
```

**تعريف "جاهز":** شركة تجارة صغيرة تقدر تبيع وتشتري وتدفع وتراجع أرقامها بشكل مستقل.

## 7. 🏆 الميزات التنافسية (Differentiators)

### ما يجعل النظام مختلفاً عن المنافسين (ODOO, SAP, Metabase):

| الميزة | التفاصيل |
|---|---|
| **سهولة الإعداد** | Onboarding يكتمل في 15 دقيقة (منافسونا: أيام/أسابيع) |
| **سعر مناسب للSMB** | اشتراك شهري بدلاً من ترخيص سنوي ضخم |
| **ZATCA Native** | ليس Add-on، بل مدمج من اليوم الأول |
| **واجهة عربية حقيقية** | RTL صحيح، مصطلحات خليجية، ليس ترجمة آلية |
| **Multi-Company** | إدارة 10 شركات من حساب واحد |
| **Offline POS** | نقطة البيع تعمل بدون إنترنت |
| **AI Forecasting** | توقع نفاد المخزون قبل 2 أسبوع |
